# Daily Summary — 2026-02-23

## Key highlights
- Root cause of Festr's AMD Turin slowdown found: NCCL v2.28.3 hardcodes AMD BW at 16 GB/s for all AMD CPUs; Turin xGMI3 actual bandwidth is 192–256 GB/s — 12-16x underestimate causes bad topology selection
- Fix: NCCL_P2P_LEVEL=SYS + NCCL_PROTO=LL (or NCCL_GRAPH_FILE with custom XML) brings 8-card all-reduce to ~40 GB/s bus bandwidth; now matching Grimulkan's switch-based setup (65–79 tok/s)

- Grimulkan reports 1700 tok/s aggregate on 16 cards with Kimi K2.5 (400+ concurrent prompts, FP8 KV, DCP on)
- DCP on 8 cards provides 3.6M KV cache — called "pretty insane" for long-context workloads

- SGLang DCP for GLM-4.7: PR #14194 and antgroup/sglang branch found; SGLANG_DCP=8 env var enables it
