## Key highlights
- GLM-5 FP4 GEMM race condition fix: FlashInfer PR #2716 merged; `--fp4-gemm-backend flashinfer_cutlass` now stable after clearing JIT cache (`rm -rf /cache/jit/*`); `flashinfer_cudnn` also valid and potentially faster; CUTLASS 4.4.1 included in updated Docker
- Fix confirmed by both Festr and Alex: 40+ minutes of intensive parallel testing without crash (previously crashed within 3 minutes)
- PDL bug note: the bug could propagate silent NaN/invalid results in high-concurrency without crashing — reliability issue beyond just stability
- WRX90E PCIe crash root cause found: `pcie_aspm=off pcie_port_pm=off` kernel cmdline parameters prevent "Surprise Link Down" errors; GPUs downgrading to Gen1 when idle then retraining to Gen5 triggers DPC; fix is to disable ASPM + port runtime PM + set `DynamicPowerManagement=0`
- After applying PCIe fix: chisleu ran 5+ minutes of 100k-token concurrent prefill stress test without crash (previously crashed in seconds at 150% load)
- Pinchbench (openclaw benchmark) results: MiniMax M2.5 and GLM-5 both under 40% success rate; Kimi K2.5 slightly behind M2.5; M2.1 scores 93.6% (best local); Opus 4.6 at 90.6%; Haiku above Opus raises questions about benchmark validity

- vLLM TTFT issue: Qwen3.5 27B shows 12.56s TTFT vs llama.cpp 0.13s; generation speed similar (~69–80 tok/s); root cause appears to be vLLM warmup/cuda graph overhead, not generation quality
