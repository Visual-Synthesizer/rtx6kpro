# Daily Summary — 2026-03-02

## Key highlights
- FlashInfer SM120f support: PR #2650 merged adding `gen_fp4_quantization_sm120f_module`; key insight — SM120 requires compilation for `sm120f` to access `cvt.rn.satfinite.e2m1x2.f32` PTX instruction; without it NVFP4 underperforms INT4
- Benchmark result: FlashInfer with CUDA 13.1 + SM120f JIT gives 65 tok/s vs compiled vLLM internal cutlass at 67 tok/s — cutlass still marginally faster for SM120
- FlashInfer PR #2460 merged: doubles NVFP4 throughput on some tiles; worth testing
- Qwen3.5 NVFP4 (397B) fp8 KV cache: `fp8` and `fp8_e4m3` are equivalent; `bfloat16` not valid as KV dtype in vLLM (use `bfloat16` flag not `bf16`)
- AlexGS discovers that setting `--kv-cache-dtype fp8` explicitly (vs auto) and disabling MTP fixes Qwen3.5 instruction-following failures — possible bug when `auto` selects wrong dtype with MTP
- GLM-5 AWQ: not recommended; NVFP4 considered superior
- chisleu hard crash recurrence on WRX90 under concurrent vLLM requests — system unresponsive to power button; BIOS update did not fix; memtest86 recommended
